# لعبة الغولف — دليل تشغيل الموقع مع تسجيل الدخول والحفظ الدائم

هذا الدليل يشرح كيف تفعّل تسجيل الدخول بالبريد الإلكتروني وكلمة المرور، وحفظ مسار كل مستخدم، وكيف تنشر الموقع على دومين خاص بك.

---

## الخطوة ١: إنشاء قاعدة بيانات مجانية على Supabase

1. اذهب إلى https://supabase.com وأنشئ حسابًا مجانيًا (يكفي بريد إلكتروني أو حساب GitHub).
2. اضغط **New Project**، واختر اسمًا (مثلاً `ghiras-golf`) وكلمة مرور لقاعدة البيانات.
3. انتظر دقيقة حتى يجهّز المشروع.
4. من القائمة الجانبية اذهب إلى **SQL Editor** → **New query**، والصق فيه محتوى ملف `schema.sql` المرفق، ثم اضغط **Run**. هذا ينشئ الجدول الذي يحفظ مسار كل مستخدم مرتبطًا بحسابه.
5. من القائمة الجانبية اذهب إلى **Project Settings → API**. ستحتاج قيمتين:
   - **Project URL** (يبدأ بـ `https://xxxxx.supabase.co`)
   - **anon public key** (سلسلة طويلة تبدأ عادة بـ `eyJ...`)

## الخطوة ٢: إعدادات تسجيل الدخول بالبريد وكلمة المرور

تسجيل الدخول بالبريد وكلمة المرور مفعّل افتراضيًا في Supabase. القرار الوحيد الذي يهمك:

**هل تريد تفعيل تأكيد البريد الإلكتروني عند التسجيل؟**
- من **Authentication → Providers → Email**، يوجد خيار **Confirm email**.
- **مفعّل (الافتراضي)**: بعد إنشاء الحساب، يصل للمستخدم بريد تفعيل يجب أن يضغط رابطه قبل أول تسجيل دخول. أكثر أمانًا.
- **معطّل**: يدخل المستخدم مباشرة بعد إنشاء الحساب دون أي بريد تفعيل. أسرع لكن أقل تحققًا من صحة البريد.

لأي من الخيارين، الموقع يتعامل معه تلقائيًا (يعرض شاشة "فعّل حسابك" إذا كان التأكيد مطلوبًا).

من **Authentication → URL Configuration**، ضع رابط موقعك النهائي في **Site URL** و **Redirect URLs** (هذا يُستخدم فقط عند "نسيت كلمة المرور" أو رابط تأكيد البريد).

## الخطوة ٣: ربط الموقع بالقاعدة

افتح ملف `index.html` وابحث عن هذا السطر بالقرب من بداية كود JavaScript:

```js
const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';
```

استبدل القيمتين بالقيم التي نسختها من الخطوة ١، مثلًا:

```js
const SUPABASE_URL = 'https://abcxyz123.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
```

احفظ الملف. بدون هذه الخطوة سيعمل الموقع بلا تسجيل دخول ولا حفظ دائم (سترى تحذيرًا أصفر أعلى الصفحة).

## الخطوة ٤: كيف تبدو تجربة الزائر

1. يفتح الموقع → يظهر له نموذج "تسجيل الدخول" مع رابط "أنشئ حسابًا" أسفله.
2. أول زيارة: يضغط "أنشئ حسابًا"، يكتب بريده وكلمة مرور (٦ أحرف على الأقل)، يضغط "إنشاء الحساب".
3. إذا كان تأكيد البريد مفعّلًا في Supabase، يرى شاشة "فعّل حسابك" ويحتاج فتح رسالة التفعيل من بريده أولًا.
4. بعد ذلك، يسجّل الدخول ببريده وكلمة المرور من أي جهاز ليستعيد مساره كاملًا.
5. يوجد رابط "نسيت كلمة المرور؟" يرسل رابط إعادة تعيين إلى البريد.
6. زر "تسجيل الخروج" أعلى الصفحة لإنهاء الجلسة.

> ملاحظة: إن استخدمت الموقع مع طلاب برنامج غراس، يُفضّل أن يستخدم كل طالب بريد والديه أو بريدًا مدرسيًا مُشرفًا عليه، خصوصًا مع وجود كلمة مرور فعلية الآن.

## الخطوة ٥: نشر الموقع على دومين خاص

أسهل خيارين مجانيين وسريعين:

**الخيار أ — Netlify**
1. اذهب إلى https://app.netlify.com
2. اسحب مجلد الموقع (يحتوي فقط على `index.html`) مباشرة إلى صفحة Netlify (Drag & Drop).
3. سيعطيك رابطًا فوريًا مثل `xyz123.netlify.app`.
4. ارجع للخطوة ٢ وحدّث Site URL / Redirect URLs في Supabase بهذا الرابط الجديد.
5. من إعدادات الموقع (Domain settings) يمكنك ربط دومينك الخاص إن كان لديك واحد.

**الخيار ب — GitHub Pages**
1. ارفع ملف `index.html` إلى مستودع (repository) جديد على GitHub.
2. من إعدادات المستودع → Pages، فعّل النشر من الفرع الرئيسي (main).
3. سيصبح الموقع متاحًا على رابط مثل `username.github.io/repo-name`.
4. حدّث Site URL / Redirect URLs في Supabase بهذا الرابط.

كلا الخيارين لا يحتاجان أي خبرة برمجية إضافية، فالموقع بأكمله ملف HTML واحد.

## الخطوة ٦: (اختياري) مراجعة المستخدمين كمشرف

- **Authentication → Users** في Supabase: قائمة كل من أنشأ حسابًا، مع إمكانية حذف أو تعطيل أي حساب.
- **Table Editor → golf_sessions**: مسار كل مستخدم كاملًا (المجالات، القرارات، الهدف النهائي) — مفيد لمتابعة مسارات طلاب برنامج غراس كمشرف.
